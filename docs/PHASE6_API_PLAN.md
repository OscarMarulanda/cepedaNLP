# Phase 6: MCP Server + Streamlit Frontend — Implementation Plan

## Context
The RAG system is complete (`src/rag/` — 10 speeches, 131 chunks). This phase exposes it via an MCP server and builds a Streamlit chat frontend where Claude (Haiku) orchestrates tool calls. No REST API — Claude decides what tools to call based on the user's natural language input.

## Architecture

```
Browser (auto-generated by Streamlit)
    │
    ▼ WebSocket
Streamlit Python process
    │
    ├──► Anthropic API (Haiku)
    │        │
    │        │ tool_use: retrieve_chunks, list_speeches, etc.
    │        │
    │        ▼
    └──► MCP Server (FastMCP, HTTP/SSE)
              │
              ▼
         PostgreSQL + pgvector
```

**Key design:** Claude is the orchestrator. MCP tools are pure data fetchers (no LLM calls). The system prompt on the Streamlit side controls personality, citation format, and constraints. This makes the MCP server reusable — it can be connected to Claude Desktop or any other MCP client.

## MCP Tools (6 tools, pure data fetchers)

| Tool | Params | Returns |
|------|--------|---------|
| `retrieve_chunks` | `query: str, top_k: int = 5` | Embedded query → pgvector search → chunks with speech title, date, YouTube timestamp link, similarity score. NO Claude call. |
| `list_speeches` | — | All speeches: id, title, date, location, event, word_count, youtube_url |
| `get_speech_detail` | `speech_id: int` | Single speech + entity_count + chunk_count + duration |
| `search_entities` | `entity_text?: str, entity_label?: str` | Entities matching search across all speeches with mention counts |
| `get_speech_entities` | `speech_id: int` | All entities in one speech grouped by label |
| `get_corpus_stats` | — | Total speeches, words, entities, annotations, chunks |

All tools use parameterized SQL. All are read-only.

## Streamlit Frontend Flow

1. User types message in `st.chat_input`
2. Streamlit calls Claude API (Haiku) with:
   - System prompt (Spanish, personality/citation rules, "answer only from tool results")
   - Conversation history (multi-turn)
   - Tool definitions matching the 6 MCP tools
3. If Claude returns `tool_use` blocks → Streamlit's MCP client calls the MCP server
4. Tool results sent back to Claude as `tool_result` blocks
5. Claude generates final answer → displayed via `st.chat_message`
6. Sources rendered with YouTube links

## Security (no auth)

| Layer | Where | How |
|-------|-------|-----|
| Input validation | MCP tool params | Pydantic via FastMCP: `max_length=500` on queries, `ge=1` on IDs |
| Prompt injection | System prompt + tool validation | System prompt constrains Claude; `retrieve_chunks` validates query with regex filter |
| SQL injection | MCP tools | Parameterized queries (`%s` placeholders) — already in place |
| Cost control | Streamlit session | Per-session message counter + cooldown. Claude only called from server-side Python — API key never exposed to browser |
| Output safety | Streamlit | `st.markdown` with controlled rendering; no raw HTML injection possible |
| Abuse | Streamlit session | Session-based rate limiting (e.g., 30 messages/session, 10/minute) |

## Key Reused Code

## How to Run
```bash
source venv/bin/activate

# Terminal 1: MCP server
python -m src.mcp.server
# Starts FastMCP HTTP server on port 8000

# Terminal 2: Streamlit frontend
streamlit run src/frontend/app.py
# Opens browser at localhost:8501
```

## Bonus: Claude Desktop Integration
Since the MCP server is standalone, it can also be added to Claude Desktop's `claude_desktop_config.json`:
```json
{
  "mcpServers": {
    "cepeda-nlp": {
      "url": "http://localhost:8000/mcp"
    }
  }
}
```
This lets you query Cepeda's speeches from Claude Desktop — a nice MVP demo.
